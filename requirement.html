1 Project overview

Project assignment 2 consists of three parts,

  1. in the assignment 2a you play the role of a client who is in need of a new website,

  2. in the assignment 2b you play the role of the web developer(s), and

  3. in the assignment 2c you continue playing the role of the web developer(s) for improving the website.

The assignment 2 is graded in four parts:

Parts Grade percentage
Assignment 2a: business description, website requirements, website content 20%
Assignment 2b: implementation of the website 50%
Assignment 2b: communication performance (on GitHub Issues) 10%
Assignment 2c: improvement of the website 20%

2 Get prepared

  1. Find (a) team partner(s). Team up with other(s) from the course and announce on the forum. IMPORTANT! There is a set of rules for team partnering, please read them carefully, they are flexible enough but have strict deadlines if you want to change your partner(s).

  2. Setup your GitHub repository. For assignment 2a, 2b and 2c it is mandatory to use the code repository service GitHub for creating an open source repo that allows sharing your deliverables and publishing the website online. IMPORTANT! Learn the correct way to setup and use GitHub in this course before you get started with your assignment. This is also important for your grade, since your contributions and communication will be tracked during grading.

  3. Choose a business. You can either choose an existing organisation if it doesn't have a website yet, or, alternatively, you can make up (invent) an organisation for this assignment. The business may be a small company in an area to which you feel related or are familiar; e.g. selling special equipment or offering special services; a sport club, music club, etc.; an environmental group, charity or social welfare association.

3 Develop a business description

Write up a business description that helps your web developer(s) to understand your organisation, including what it does and how it wants to be portrayed to the public. The description must be succinct and clear, and should help

  • you, as the client, to understand your own project, and,

  • your developer(s) in making design decisions.

The business description should include:

  1. The official name of your business. (*)

  2. Where the business office(s) is/are located, including the address of the headquarter(s) and branches (for a potential "Contact" page). Even if the business is a purely online business, it needs to show a physical presence in order to create credibility. (*)

  3. Contact information: email, phone numbers, social media accounts, etc. (*).

  4. A brief portrait of your business for an "About" page. This should include a description of its mission. (*)

  5. Product, services and/or business activities, describing how they are to be supported by the new website. (*)

  6. User personas of the main users/customers of the new website. A user persona is a semi-fictional character created based on your knowledge of your current, or ideal, user/customer. Create two user personas distinguishing clearly two different user profiles, based on behaviour, interests, needs, desires, etc. using this guide on personas and this guide on user profiles. (*)

  7. Any other relevant information that helps the web developer(s).

* Mandatory items.

4 Elaborate website requirements

  1. Make a list of all webpages. Including at least 7/ 10/13 webpages for a team of 1/2/3 members, and at most 20. For each planned webpage provide

    • a file name,

    • a title (for the title element),

    • a description of the text and pictures/images to include, and

    • a description of the functionality, if pertinent.

    This list of four columns should be delivered in the form of a table in an HTML file.

  2. Make a website design diagram. Specify the website's views, areas and pages using the method described in the lecture slides about Website Design. Create the website design diagram using Balsamiq, a tool for creating wireframes online and collaboratively, and that allows you to export your design to PDF format files.

  3. Make sure to include 2/3/4 of the following items for a team of 1/2/3 members on your website:

    • a page based on an XML data file, such as an address list or a product list,

    • one or more SVG images, such as a logo, icons, decorative graphics, etc.,

    • audio or video files or web links to streaming services. Find instructions for embedding YouTube, Twitch, Vimeo, SoundCloud, Spotify, Deezer, etc.

  4. Elaborate visual style. Delineate how the website should look and feel: colors, fonts, layout, decorative elements, etc.

5 Prepare website content

  1. Choose a domain name, and share the link to the domain checker showing that the domain is still available. Go to Name.com, find a suitable domain available, and include the link in your assignment submission. Look at this example of a link to a domain checker showing availability. There is no need to buy a domain, but you are free to do it if you want.

  2. Create text, enough to fill up all the pages with at least two paragraphs with realistic text, avoiding always pages that uniquely contain images; if not centric, text must be an important part of every web page. Therefore, never use placeholder (fake or filler) text.

  3. Create or collect images, read our guide to use images on the Web and learn how to properly get and use images in your project. You must provide credits for all images, stating which license applies to which image, either in the footer of the page where the image is used, or on a special credits.html page.

  4. Provide the business logo. Consider to design a logo for free with Adobe Spark. The logo may be provided as an image file, to be marked up as an SVG file, being an item of 4c.

  5. Provide files/data/web links of items to markup/embed in 4c.

IMPORTANT!: your website should not include interactive elements, such as forms or widgets. Therefore, you should disregard e-commerce/app-alike features and focus exclusively on content presentation and consumption.

6 Deliverables

You submit the link to the (centralised and unique) group's GitHub repo, which must include the following:

  1. README.md, with information of the web developer(s), such as names & GitHub usernames, page with business description, website requirements and website content.

  2. Create individual folders for each assignment part: "assignment-2a", "assignment-2b", "assignment-2c".

  3. In your folder assignment-2a, create a subfolder website-content and include

    • a list of all webpages, webpages-list.html.

    • a website design diagram, website-design-diagram.pdf, generated with Balsamiq.

    • data/images/video/audio files, as defined in 4c.

For this assignment there is no need for individual submissions, since assignment submission has been set for group submission, therefore it suffices if only one member submits a group's work.
